How CryptoGuard Works

Understanding the 6-layer ML pipeline

1

Data Collection

Real market data from CoinGecko (OHLCV, market cap) and DeFiLlama (TVL, protocol metrics). No synthetic or proxy features — all data is real and honestly labeled.

CoinGecko APIDeFiLlama API
2

Anomaly Detection

K-Means Clustering identifies market states (Calm, Volatile, Stressed). Isolation Forest computes anomaly scores to flag unusual market conditions.

K-MeansIsolation Forest
3

Market Regime Detection

Gaussian HMM discovers hidden market states. XGBoost classifies regimes into Bull, Bear, or Sideways. A blended score combines both models (α = 0.45).

HMMXGBoost
4

Airdrop Prediction

Protocol data from DeFiLlama is fed into Logistic Regression and XGBoost classifiers (with SMOTE) to predict airdrop probability.

Logistic RegressionSMOTE
5

Decision Advisory

A composite risk score combines anomaly intensity (40%), regime risk (35%), drawdown (15%), and volatility (10%). This maps to a 6-tier capital allocation advisory.

Risk Score6-Tier Advisory
6

Deployment

The entire pipeline is served via a FastAPI REST API and this interactive web dashboard. All data is updated by re-running the pipeline scripts.

FastAPIChart.js
